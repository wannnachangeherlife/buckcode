## 🗂️ 数据结构与同步策略

### **1. Notion 主数据库设计**

#### 1.1 课程表（Course Table）
```
属性：
- 课程名称 [Title]
- 学科分类 [Category]: AI算法/3D建模/前端/后端/数学/英语/考试
- 开始日期 [Start Date]
- 结束日期 [End Date]
- 周期 [Recurrence]: 一周几小时
- 优先级 [Priority]: 高/中/低
- 状态 [Status]: 未开始/进行中/已完成/暂停
- 学习资源链接 [Resources]: URL列表
- 关联任务 [Relation]: Task Table
- 备注 [Notes]
```

#### 1.2 任务清单（Task Table）
```
属性：
- 任务标题 [Title]
- 任务类型 [Type]: 讲座/作业/项目/复习/考试/阅读
- 截止日期 [Due Date] ⭐ 关键字段
- 预计耗时 [Estimated Hours]
- 实际耗时 [Actual Hours]
- 优先级 [Priority]
- 状态 [Status]: 未开始/进行中/已完成/延期/取消
- 关联课程 [Course Relation]
- 完成度 [Progress]: 0-100%
- 提交物 [Deliverables]: 代码/笔记/报告/视频/截图
- 提交链接 [Submission Link]: GitHub Gist/链接
- AI反馈需求 [AI Feedback]: True/False
- 复习周期 [Review Schedule]: D+1/D+3/D+7/D+14/D+30
- 笔记链接 [Note Link]: Obsidian URI
- 标签 [Tags]: #考试周 #突发 #延期 #加急
```

#### 1.3 学习进度追踪（Progress Table）
```
属性：
- 日期 [Date]
- 学习课程 [Course Relation]
- 学习时长 [Duration Hours]
- 学习内容 [Content]
- 效率评分 [Efficiency]: 1-10
- 学习形式 [Method]: 视频/阅读/编码/讨论/实验
- 笔记数 [Note Count]
- 问题数 [Questions Count]
- 完成任务数 [Tasks Completed]
- 心得/反思 [Reflection]
- 截图/证明 [Evidence]: 截图或代码提交链接
```

#### 1.4 模板库（Template Table）
```
属性：
- 模板名称 [Title]
- 模板类型 [Type]: 周计划/日计划/学习笔记/费曼讲解/复习卡片/项目提案
- 模板内容 [Content]: 预制的Markdown/结构化问题
- 使用频率 [Usage]
- 最后使用日期 [Last Used]
```

#### 1.5 考试与工作事件（Events Table）
```
属性：
- 事件名称 [Title]
- 事件类型 [Type]: 考试/工作截止/会议/假期/重要日期
- 日期 [Date]
- 持续时间 [Duration]
- 优先级 [Priority]: 极高/高/中/低
- 相关课程 [Related Courses]
- 准备进度 [Prep Progress]
- 提醒时间 [Reminders]: -7天/-3天/-1天/当天上午/当天
- 备注 [Notes]
```

#### 1.6 资源库（Resource Table）
```
属性：
- 资源名称 [Title]
- 资源类型 [Type]: 教程/论文/代码/工具/数据集/视频
- URL/路径 [Link]
- 相关课程 [Courses]
- 优先级 [Priority]
- 阅读状态 [Status]: 未读/阅读中/已读/已归档
- 笔记链接 [Notes]
- 标签 [Tags]
```

### **2. 同步流向与触发机制**

```
Notion 任务创建/更新
    ↓ (trigger)
Python脚本检测变化
    ├─→ 提取截止日期 & 优先级
    ├─→ 生成GitHub Issue
    ├─→ 创建Apple日历事件
    ├─→ 发送邮件通知
    └─→ 更新Obsidian Daily Note
    
GitHub Issue 完成
    ↓ (webhook)
自动更新Notion Task状态
    ├─→ 关闭对应Issue
    ├─→ 记录完成时间
    └─→ 触发AI反馈流程

Apple日历 任务提醒触发
    ↓ (iOS Shortcut)
推送通知到iPhone/iPad
    ├─→ 快速打开Notion
    ├─→ 更新任务状态
    └─→ 记录开始时间
```

## 🔄 核心自动化脚本需求

### **脚本1: Notion-GitHub同步脚本**
```python
# 功能
- 每小时检测Notion新任务
- 自动创建GitHub Issue（标签、优先级、里程碑）
- Issue完成时，自动更新Notion状态
- 生成每周任务统计报告
```

### **脚本2: 定时提醒与日历同步脚本**
```python
# 功能
- 每天早上6:30发送邮件摘要（今日任务+昨日完成情况）
- 将Notion任务同步到Apple日历（需Zapier或直接API调用）
- D+1/D+3/D+7/D+14自动提醒复习
- 生成Obsidian Daily Note模板
```

### **脚本3: AI反馈生成脚本**
```python
# 功能
- 从Notion提取标记为"需AI反馈"的任务
- 调用ChatGPT API生成针对性反馈
- 保存反馈到Notion & Obsidian
- 生成个性化改进建议
```

### **脚本4: 学习统计与可视化脚本**
```python
# 功能
- 汇总Progress表数据
- 生成周/月学习报告（课程分布、时长统计、效率趋势）
- 输出JSON供Figma可视化面板
- 识别低效课程、需加强区域
```

## 🔐 数据安全与隐私

- **Notion**: 企业认证Account

- **Apple日历**: 本地存储，不上传云端
- **.env文件**: 存储所有API密钥，加入.gitignore
- **自动化脚本**: 仅本地运行，不上传GitHub

## 📚 工具间的数据格式统一

- P0（极高）→ GitHub Label:P0 | Notion:🔴

- P3（低）→ GitHub Label:P3 | Notion:🟢

- Notion状态 → GitHub状态映射
  - 未开始 → Open
  - 进行中 → In Progress
  - 已完成 → Closed
  - 延期 → On Hold
  - 取消 → Cancelled
